# Claude Code Instructions

## プロジェクトルール

このプロジェクトでコードを生成・修正する際は、以下のルールを必ず遵守してください。

## 必須チェック

**コードを変更した後は、必ず以下を実行してください：**

### Markdown linterチェック

```bash
make lint
```

### すべてのチェック

```bash
make check
```

## 開発原則

- ドメイン駆動設計を前提に設計
- テスト駆動開発（TDD）を心がける
- セキュリティとパフォーマンスを最大限考慮

## Markdownファイルの規約

- markdownlintのルールに準拠
- コードブロックには言語指定を必ず含める
- リストの前後には空行を入れる
- ファイルの末尾には改行を入れる

## コミット前の必須チェック

コードを変更した後、コミットする前に：

```bash
make lint
```

エラーが検出された場合は、`make lint-fix` で自動修正を試み、再度チェックしてください。
